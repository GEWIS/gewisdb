MailingListMember.php : 29

Mailing List Member model.

To allow having additional properties in the many-to-many association between MailingLists and Members we use this class as a connector.

Mailing list <-> member associations are never directly propagated to Mailman. When synchronizing the state directly, the chances of something going wrong are too high. For example, we do not want someone to be registered in Mailman for a list, but this is not directly visible in the database. What we do want is to always know in the database the state of a member who is a member of a mailing list, as such persisting this entity is our highest priority.

The actual synchronization should take place through cron jobs. To keep track of what is supposed to happen, the additional properties in this entity are used for this.
